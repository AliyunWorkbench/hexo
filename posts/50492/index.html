A*

在探讨 A∗A^{\ast}A∗ 算法之前, 我们先来回顾一下优先队列 BFS 算法。该算法维护了一个优先队列 (二叉堆), 不断从堆中取出 “当前代价最小” 的状态 (堆顶) 进行扩展。每个状态第一次从堆中被取出时, 就得到了从初态到该状态的最小代价。

如果给定一个 “目标状态”, 需要求出从初态到目标状态的最小代价, 那么优先队列 BFS 的这个 “优先策略” 显然是不完善的。一个状态的当前代价最小, 只能说明从起始状态到该状态的代价很小, 而在未来的搜索中, 从该状态到目标状态可能会花费很大的代价。另外一些状态虽然当前代价略大, 但是未来到目标状态的代价可能会很小, 于是从起始状态到目标状态的总代价反而更优。优先队列 BFS 会优先选择前者的分支, 导致求出最优解的搜索量增大。比如在优先队列 BFS 的示意图中, 产生最优解的搜索路径 (5+2+1)(5+2+1)(5+2+1) 的后半部分就很晩才得以扩展。

为了提高搜索效率, 我们很自然地想到, 可以对未来可能产生的代价进行预估。详细地讲, 我们设计一个 “估价函数”, 以任意 “状态” 为输入, 计算出从该状态到目标状态所需代价的估计值。在搜索中, 仍然维护一个堆, 不断从堆中取出 “当前代价+未来估价” 最小的状态进行扩展。

为了保证第一次从堆中取出目标状态时得到的就是最优解, 我们设计的估价函数需要满足一个基本准则:

  • 设当前状态 state 到目标状态所需代价的估计值为 f\mathrm{f}f (state)。
  • 设在未来的搜索中, 实际求出的从当前状态 state 到目标状态的最小代价为 g\mathrm{g}g (state)。
  • 对于任意的 state, 应该有 f\mathrm{f}f (state) ≤g\leq \mathrm{g}≤g (state)。

也就是说, 估价函数的估值不能大于未来实际代价, 估价比实际代价更优。为什么要遵守这个准则呢? 我们不妨看看, 如果某些估值大于未来实际代价, 那么将发生什么情况。

我们可以看到, 本来在最优解搜索路径上的状态被错误地估计了较大的代价, 被压在堆中无法取出, 从而导致非最优解搜索路径上的状态不断扩展, 直至在目标状态上产生错误的答案。

如果我们设计估价函数时遵守上述准则, 保证估值不大于未来实际代价, 那么即使估价不太准确, 导致非最优解搜索路径上的状态 sss 先被扩展, 但是随着 “当前代价” 的不断累加, 在目标状态被取出之前的某个时刻:

  1. 根据 sss 并非最优, sss 的 “当前代价” 就会大于从起始状态到目标状态的最小代价。
  2. 对于最优解搜索路径上的状态 ttt, 因为 f(t)≤g(t)\mathrm{f}(t) \leq \mathrm{g}(t)f(t)≤g(t), 所以 ttt 的 “当前代价” 加上 f(t)\mathrm{f}(t)f(t) 必定小于等于 ttt 的 “当前代价” 加上 g(t)\mathrm{g}(t)g(t), 而后者的含义就是从起始状态到目标状态的最小代价。

结合以上两点, 可知 “ ttt 的当前代价加上 f(t)\mathrm{f}(t)f(t) ” 小于 sss 的当前代价。因此, ttt 就会被从堆中取出进行扩展, 最终更新到目标状态上, 产生最优解。

这种带有估价函数的优先队列 BFS 就称为 A∗A^{\ast}A∗ 算法。只要保证对于任意状态 state, 都有 f(state)≤g(state)\mathrm{f}( state ) \leq \mathrm{g} (state)f(state)≤g(state) , A∗A^{\ast}A∗ 算法就一定能在目标状态第一次从堆中被取出时得到最优解。 (~~并且在搜索过程中每个状态只需要被扩展一次 (之后再被取出就可以直接忽略)~~中间点可能被扩展（入队）多次，中间点第一次出队不一定是最短距离，只有终点第一次出队可保证是最短距离)。估价 f\mathrm{f}f (state) 越准确、越接近 g\mathrm{g}g (state), A∗\mathrm{A}^{\ast}A∗ 算法的效率就越高。如果估价始终为 0 , 就等价于普通的优先队列 BFS。

A∗\mathrm{A}^{*}A∗ 算法提高搜索效率的关键, 就在于能否设计出一个优秀的估价函数。估价函数在满足上述设计准则的前提下, 还应该尽可能反映未来实际代价的变化趋势和相对大小关系, 这样搜索才会较快地逼近最优解。接下来我们通过两道例题来具体地感受一下 A∗A^{*}A∗ 算法。

178. 第K短路

题目描述

给定一张 NNN 个点（编号 1,2…N1,2…N1,2…N），MMM 条边的有向图，求从起点 SSS 到终点 TTT 的第 KKK 短路的长度，路径允许重复经过点或边。

注意： 每条最短路中至少要包含一条边。

输入格式

第一行包含两个整数 NNN 和 MMM。

接下来 MMM 行，每行包含三个整数 A,BA,BA,B 和 LLL，表示点 AAA 与点 BBB 之间存在有向边，且边长为 LLL。

最后一行包含三个整数 S,TS,TS,T 和 KKK，分别表示起点 SSS，终点 TTT 和第 KKK 短路。

输出格式

输出占一行，包含一个整数，表示第 KKK 短路的长度，如果第 KKK 短路不存在，则输出 −1-1−1。

数据范围

1≤S,T≤N≤10001 \le S,T \le N \le 10001≤S,T≤N≤1000,
0≤M≤1050 \le M \le 10^50≤M≤105,
1≤K≤10001 \le K \le 10001≤K≤1000,
1≤L≤1001 \le L \le 1001≤L≤100

算法分析

一个比较直接的想法是使用优先队列 BFS 进行求解。优先队列 (堆) 中保存一些二元组 (x,dist)(x, d i s t)(x,dist), 其中 xxx 为节点编号, dist 表示从 SSS 沿着某条路径到 xxx 的距离。

起初, 堆中只有 (S,0)(S, 0)(S,0) 。我们不断从堆中取出 dist 值最小的二元组 (x,dist)(x, d i s t)(x,dist), 然后沿着从 xxx 出发的每条边 (x,y)(x, y)(x,y) 进行扩展, 把新的二元组 (y,dist+length(x,y))(y, d i s t+l e n g t h(x, y))(y,dist+length(x,y)) 插入到堆中 (无论堆中是否已经存在一个节点编号为 yyy 的二元组)。

上一节我们已经讲到, 在优先队列 BFS 中, 某个状态第一次从堆中被取出时, 就得到了从初态到它的最小代价。读者用数学归纳法很容易得到一个推论: 对于任意正整数 iii 和任意节点 xxx, 当第 iii 次从堆中取出包含节点 xxx 的二元组时, 对应的 dist 值就是从 SSS 到 xxx 的第 iii 短路。(所以, 当扩展到的节点 yyy 已经被取出 KKK 次时, 就没有必要再插入堆中了。注意：如果使用 A∗A^{\ast}A∗ 算法，加了估价函数后，中间的第一次出队不一定是最短距离，故不能只用取出 KKK 次进行优化)。最后当节点 TTT 第 KKK 次被取出时, 就得到了 SSS 到 TTT 的第 KKK 短路。

使用优先队列 BFS\mathrm{BFS}BFS 在最坏情况下的复杂度为 O(K∗(N+M)∗log⁡(N+M))O(K *(N+M) * \log (N+M))O(K∗(N+M)∗log(N+M)) 。这道题目给定了起点和终点, 求长度最短 (代价最小) 的路径, 可以考虑使用 A∗\mathrm{A}^{*}A∗ 算法提高搜索效率。

根据估价函数的设计准则, 在第 KKK 短路中从 xxx 到 TTT 的估计距离 f(x)\mathrm{f}(x)f(x) 应该不大于第 KKK 短路中从 xxx 到 TTT 的实际距离 g(x)\mathrm{g}(x)g(x) 。于是, 我们可以把估价函数 f(x)\mathrm{f}(x)f(x) 定为从 xxx 到 T\mathrm{T}T 的最短路长度, 这样不但能保证 f(x)≤g(x)\mathrm{f}(x) \leq \mathrm{g}(x)f(x)≤g(x), 还能顺应 g(x)\mathrm{g}(x)g(x) 的实际变化趋势。 最终我们得到了以下 A∗A^{*}A∗ 算法:

  1. 预处理出各个节点 xxx 到终点 TTT 的最短路长度 f(x)\mathrm{f}(x)f(x) 一一这等价于在反向图上以 TTT 为起点求解单源最短路径问题, 可以在 O((N+M)log⁡(N+M))\mathrm{O}((N+M) \log (N+M))O((N+M)log(N+M)) 的时间内完成。
  2. 建立一个二叉堆, 存储一些二元组 (x,dist+f(x))(x, dist +\mathrm{f}(x))(x,dist+f(x)), 其中 xxx 为节点编号, dist 表示从 SSS 到 xxx 当前走过的距离。起初堆中只有 (S,0+f(0))(S, 0+\mathrm{f}(0))(S,0+f(0)) 。
  3. 从二叉堆中取出 dist+f(x)dist +\mathrm{f}(x)dist+f(x) 值最小的二元组 (x,dist+f(x))(x, dist +\mathrm{f}(x))(x,dist+f(x)), 然后沿着从 xxx 出发的每条边 (x,y)(x, y)(x,y) 进行扩展。如果节点 yyy 被取出的次数尚未达到 KKK , 就把新的二元组 (y,dist+length(x,y)+f(y))(y,dist + length (x, y)+\mathrm{f}(y))(y,dist+length(x,y)+f(y)) 插入堆中。
  4. 重复第 2∼32 \sim 32∼3 步, 直至第 KKK 次取出包含终点 TTT 的二元组, 此时二元组中的 dist 值就是从 SSS 到 TTT 的第 KKK 短路。

A∗A^{*}A∗ 算法的复杂度上界与优先队列 BFSB F SBFS 相同。不过, 因为估价函数的作用, 图中很多节点访问次数都远小于 KKK, 上述 A∗A^{*}A∗ 算法已经能够比较快速地求出结果。

算法分析

先进行可解性判定。我们在排序-逆序对中已经提到过, 把除空格之外的所有数字排成一个序列, 求出该序列的逆序对数。如果初态和终态的逆序对数奇偶性相同, 那么这两个状态互相可达, 否则一定不可达。

若问题有解, 我们就采用 A∗A^{*}A∗ 算法搜索一种移动步数最少的方案。经过观察可以发现, 每次移动只能把一个数字与空格交换位置, 这样至多把一个数字向它在目标状态中的位置移近一步。即使每一步移动都是有意义的, 从任何一个状态到目标状态的移动步数也不可能小于所有数字当前位置与目标位置的曼哈顿距离之和。

于是, 对于任意状态 state, 我们可以把估价函数设计为所有数字在 state 中的位置与目标状态 end 中的位置的曼哈顿距离之和, 即:

f(state)=∑num=19(∣state_xnum−end_xnum∣+∣state_ynum−end_ynum∣)f(state)=\sum_{num=1}^{9}\left(\mid state\_x_{n u m}-end\_ x_{num}|+| state\_y_{num} - end\_y_{num} \mid\right) f(state)=num=1∑9​(∣state_xnum​−end_xnum​∣+∣state_ynum​−end_ynum​∣)

其中 state_xnumstate\_x_{num}state_xnum​ 表示在状态 state 下数字 num 的行号, state_ynumstate\_y_{num}state_ynum​ 为列号。 我们不断从堆中取出“从初态到当前状态 state 已经移动的步数+f(state)” 最小的状态进行扩展, 当终态第一次被从堆中取出时, 就得到了答案。

在 A∗A^{*}A∗ 算法中, 为了保证效率, 每个状态只需要在第一次被取出时扩展一次。本题中的状态是一个八数码, 并非一个简单的节点编号, 所以需要使用 Hash 来记录每个八数码是否已经被取出并扩展过一次。我们可以选择取模配合开散列处理冲突, 或 STL map 等 Hash 方法。另外, 有一种名为 “康托展开” 的 Hash 方法, 可以对全排列进行编码和解码, 把 1∼N1 \sim N1∼N 排成的序列与 1∼N!1 \sim N !1∼N! 之间的整数建立一一映射关系, 非常适合八数码的 Hash, 感兴趣的读者可以自行查阅相关资料。
